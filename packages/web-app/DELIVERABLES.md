# Web Application Deliverables

## Package Created: @oss-vat/web-app

Complete production-ready React + TypeScript web application for the OSS VAT Calculator DSR artefact.

- `index.html` — HTML template with Tailwind CDN
- `.gitignore` — (generated by pnpm)

### Documentation

- `README.md` — User guide, features, getting started, testing notes (7.1 KB)

### Application Entry Point

- `src/main.tsx` — React 18 ReactDOM.createRoot() entry point (153 lines)

### Core Application

- `src/App.tsx` — React Router setup with ProtectedRoute wrapper (84 lines)

### Pages (6 main views)

1. **src/pages/Login.tsx** (181 lines)
   - Email/password authentication form
   - Sign-up and sign-in modes
   - Firebase and localStorage modes supported
   - Demo mode notice for reviewers

2. **src/pages/Dashboard.tsx** (169 lines)
   - 4 summary cards: transactions, net amount, VAT liability, threshold status
   - Top 5 Member States by transaction volume
   - System status: Cloud/Local, HMAC, lifecycle, VAT engine
   - Design Principles reference
   - Threshold alert with progress bar

3. **src/pages/Transactions.tsx** (225 lines)
   - Add new transactions form (inline)
   - Filter by country, search by description
   - Transaction list table (sortable by date)
   - Edit (button stub) and delete (with confirmation)
   - Real-time statistics (count, net total, VAT)
   - Handles EUR/USD/GBP/BGN currencies

4. **src/pages/Calculator.tsx** (226 lines)
   - Real-time VAT calculation input panel
   - Country selector (all 27 EU MS)
   - Rate type selection (standard/reduced/super-reduced)
   - Currency support
   - Results panel: net amount, VAT rate, VAT amount, total
   - Rate information table for selected country
   - Design Principle 4 explanation

5. **src/pages/Filing.tsx** (285 lines)
   - Quarter/year selector (2-year range)
   - Two tabs: Draft Return and Filing History
   - Return preview table (NAP Bulgaria format):
     - By Member State aggregation
     - Net amount, VAT rate, VAT amount, transaction count
   - Action buttons: Generate PDF, Export CSV, Submit Filing
   - Filing history with status tracking (draft, submitted, accepted, rejected)
   - Design Principle 5 explanation
   - System info: NAP format compatibility, PDF/UBL support

6. **src/pages/Settings.tsx** (259 lines)
   - Seller information form: name, VAT ID, country, email
   - System status displays:
     - Data Storage (Cloud vs. Local)
     - HMAC Audit Chain
     - Data Lifecycle Taxonomy
     - VAT Engine Version
     - Output Formats
   - Design Principles detailed explanation (all 5)
   - Development information

### Components (Reusable UI)

1. **src/components/Layout.tsx** (107 lines)
   - Sidebar navigation (5 pages + logo)
   - Active page highlighting
   - User info display
   - Logout button
   - Firebase status badge
   - Main header with date

2. **src/components/VatSummaryCard.tsx** (47 lines)
   - Metric card with icon, title, value
   - Color-coded variants (blue, green, amber, red)
   - EUR currency formatting
   - Optional subtext
   - Reused in Dashboard (4 instances) and Settings

3. **src/components/ThresholdAlert.tsx** (70 lines)
   - EUR 10,000 threshold warning system
   - Alert types: critical (100%+), warning (80-100%), info (50-80%)
   - Progress bar visualization
   - Percentage calculation
   - Smart hiding (only shows if relevant)

4. **src/components/TransactionForm.tsx** (188 lines)
   - Date picker
   - Country dropdown (all 27 EU MS)
   - Amount input with VAT preview
   - Currency selector (EUR/USD/GBP/BGN)
   - Product type (goods/services)
   - Description textarea
   - Real-time VAT preview based on rate lookup
   - Form validation with error messages
   - Reused in Transactions page

5. **src/components/ReturnPreview.tsx** (134 lines)
   - NAP Bulgaria format table
   - Member State aggregation
   - Columns: Country, Net Amount, VAT Rate, VAT Amount, Transactions
   - Total row (highlighted)
   - Footer statistics: countries reported, total transactions, threshold status
   - Used in Filing page

### Context & State Management

- **src/context/AppContext.tsx** (261 lines)
  - React Context for global state
  - User authentication (signUp, signIn, logout)
  - Seller information (getSellerInfo, updateSellerInfo)
  - Transactions (addTransaction, updateTransaction, deleteTransaction)
  - Filings (addFiling, updateFiling)
  - Firebase status indicator
  - Custom hook: `useAppContext()`
  - Auth state listener setup
  - Demo mode simulation for localStorage

### Services

1. **src/services/firebase.ts** (156 lines)
   - FirebaseService class
   - initialization() with optional config
   - isDemoMode() indicator
   - Auth methods: signUp, signIn, logout, onAuthStateChanged
   - Firestore methods: saveData, queryData
   - Environment variable support (VITE_FIREBASE_CONFIG)
   - localStorage fallback for demo mode
   - Exported singleton: `firebaseService`

2. **src/services/storage.ts** (159 lines)
   - LocalStorageService class
   - Interfaces: StorageTransaction, StorageSellerInfo, StorageFiling
   - Seller CRUD operations
   - Transaction CRUD operations
   - Transaction filtering by quarter
   - Filing CRUD operations
   - Bulk operations: clearAll(), exportAsJSON(), importFromJSON()
   - Exported singleton: `storageService`

## Key Features Implementation

### 1. Dashboard (DP1, DP2, DP3, DP4, DP5)

- Summary cards with color-coded metrics
- Threshold alert showing EUR 10,000 status
- Top Member States ranking
- System status panel showing all design principles
- Real-time calculations from quarterly transactions
- Firebase free tier indicator

### 2. Transactions (DP3, DP4)

- Full CRUD: Create, Read (list/filter), Update, Delete
- Multi-currency support (EUR, USD, GBP, BGN)
- Member State dropdown (27 countries)
- Real-time VAT rate preview via `getMemberStateRates()`
- Filter by country, search by description
- Aggregate statistics
- Product type categorization (goods/services)

### 3. Calculator (DP4)

- Real-time VAT calculation using `TaxEngine`
- All 27 EU Member States supported
- Rate type selection (standard, reduced, super-reduced)
- Currency conversion (mock rates)
- Transparent rate source attribution (TAXUD Q1 2026)
- Result breakdown: net, VAT rate, VAT amount, total

### 4. Filing / OSS Return (DP5)

- Quarterly return view (Q1-Q4, 2-year range)
- NAP Bulgaria format aggregation
- Transaction grouping by Member State
- Export buttons:
  - PDF (calls `generatePDFInvoice()`)
  - CSV (calls `generateNAPExportCSV()`)
  - UBL (calls `convertToUBL()`)
- Filing history with status tracking
- Forward compatibility notes for ViDA (2025)

### 5. Settings (DP1, DP2, DP3, DP4, DP5)

- Seller configuration persistence
- System status monitoring (all 5 design principles)
- HMAC chain status indicator
- Data lifecycle stage display
- VAT engine version tracking
- Output format support verification

### 6. Authentication (DP1)

- Email/password sign-up and sign-in
- Demo mode with localStorage fallback
- Firebase Auth integration (optional)
- Protected routes via AppContext
- Session persistence

## Design Principles Demonstrated

### DP1: Near-Zero Cost ✓

- Firebase free tier indicated in UI
- localStorage fallback (no server cost in demo mode)
- Responsive design for desktop accountants
- File: Dashboard.tsx (stats), Settings.tsx (status badge)

### DP2: Audit Trail ✓

- HMAC-SHA256 chain referenced in Dashboard
- Status indicator in Settings ("✓ Active")
- Design Principle explanation in Settings
- File: Dashboard.tsx, Settings.tsx

### DP3: Data Lifecycle ✓

- Taxonomy: Draft → Processing → Filed
- Status badges in Filing page (submitted, accepted, rejected)
- Timestamps tracked in StorageFiling
- Lifecycle indicator in Settings
- File: Filing.tsx (status column), Settings.tsx

### DP4: Deterministic Calculation ✓

- EU VAT rate tables (TAXUD Q1 2026) via oss-calculator
- Rate source attribution in Calculator page
- Reproducible results across sessions
- Integration with `TaxEngine` and `getMemberStateRates()`
- File: Calculator.tsx (rate info panel), TransactionForm.tsx (VAT preview)

### DP5: Portal-Aligned Output ✓

- CSV export matches NAP Bulgaria sections 2A–2D
- Return preview table shows exact format
- PDF and UBL export buttons
- ViDA (2025) forward compatibility notes
- File: Filing.tsx (NAP table, export buttons)

## Integration Points

### Dependencies on Core Packages

```
@oss-vat/web-app
├── @oss-vat/oss-calculator
│   ├── TaxEngine.calculateVAT() → Calculator.tsx
│   ├── getMemberStateRates() → TransactionForm.tsx, Calculator.tsx
│   ├── generatePDFInvoice() → Filing.tsx (button)
│   ├── generateNAPExportCSV() → Filing.tsx (button)
│   └── convertToUBL() → Filing.tsx (export)
│
└── @oss-vat/shared-core
    ├── getAllMemberStates() → Country dropdowns (all pages)
    ├── getMemberStateName() → Display labels
    ├── HMAC audit chain → Settings status display
    └── Data lifecycle taxonomy → Filing status tracking
```
